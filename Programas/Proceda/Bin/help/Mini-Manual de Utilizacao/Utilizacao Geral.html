Mini-Manual

Utilização Geral


Abrindo Arquivos

Os arquivos do Proceda são inconfundíveis pois possuem como extensão o próprio nome do programa.
Ex: "Paracatu.proceda".

Também poderá existir um arquivo associado ao arquivo "*.proceda". Este arquivo possuirá a extensão "extra" e conterá informações adicionais.
Ex: "Paracatu.proceda.extra"

Os dois arquivos deverão permanecerem sempre juntos mesmo que isto não seja necessário.



Após a abertura, as informações estarão disponívies no painel esquerdo da tela em forma de árvore hierárquica.
ATENÇÃO: Mais de um arquivo poderá estar aberto ao mesmo tempo.

Trabalhando com as informações dos arquivos



Após aberto o arquivo será criado um nó dentro da sub-pasta "Dados" e este nó receberá o nome do arquivo sem seu caminho. Dentro deste novo nó encontraremos outros sub-nós: Dados Temporais, Propriedades, Conecções e Arquivos Associados.

Dados Temporais

Este é o nó mais importante do programa, dentro dele encontraremos os nós que representam os postos. Estes dados ficam nesta pasta pois estão extremamente vinculados ao tempo.

Propriedades, Conecções e Arquivos Associados

São nós secundários, fora "Conecções" (que ainda não foi implementado), os outros nós somente mostram informações relacionadas ao arquivo e aos dados.

Ações

Como se pode notar pela imagem acima, todas as ações do programa são contextualizadas de acordo com o item da árvore selecionado. Portanto se uma ação não é mostrada ou está indisponível, é porque ela não é aplicável ao item selecionado.

ATENÇÃO: Mais de um item pode estar selecionado ao mesmo tempo.

Neste caso, somente as ações comuns aos items selecionados são mostradas.

As ações de um menu de contexto poderão estar agrupadas em ações Individuais ou Coleticas. As ações do menu principal e da barra de ferramentas são somente ações individuais.

Ações Individuais

São ações aplicadas individualmente a cada um dos itens selecionados.
Ex: Visualização em Planilha
      Será criada uma planilha individual para cada item selecionado.



Ações Coletivas

São ações que, quando aplicadas, geralmente resultam em um único resultado combinando os itens selecionados.
Ex: Visualização em Planilha
      Será criada uma única planilha que conterá os itens, por exemplo, um em cada coluna.



Importando Dados





A primeira parte da janela mostra os módulos de importação (plugins) disponíveis. Abaixo é mostrado as propriedades do módulo selecionado, estas propriedades influem na maneira como os arquivos são importados. A seguir vem os arquivos a serem importados, mesmo que a lista contenha mais do que um arquivo, estes arquivos serão importados para dentro do Proceda como um único conjunto de postos.

Geração de Séries Temporais (em Experimentação)

Uma série é obtida pela aplicação de um algorítmo que agrega/seleciona/modifica os dados de um posto.
Ex: Totais Mensais
      Série obtida sobre dados diários onde cada elemento é obtido totalizando-se os valores diários de um mês.



Análise das Séries

Gráficos Dupla-Massa e Curva de Permanencia

Sâo realizados através das ações coletivas.
No gráfico Dupla-Massa, as coordenadas Y dos pontos são fornecidas pelos valores da primeira série selecionada e as coordenadas X são fornecidas pela média dos valores encontrados nas séries restantes.

Estatísticas Descritivas

Fornece um conjunto básico de medidas de posição e percentis para todas as séries selecionadas.





Análise de Frequências

Realiza a análise de frequências para todas as séries selecionadas.




Análise de Correlações

Realiza a análise de correlações de todas as séries selecionadas. Nas próximas versões estes dados serão utilizados no preenchimento automático das falhas.


Para séries mensais, a correlação poderá ser executada mês a mês.

Banco de Dados

Criando uma Conecção

Tipos de Conecção

Até o momento, somente conecções a banco de dados padrão "Access" são permitidas.
Em versões futuras outros tipos de conecções serão permitidas.

Dados da Conecção

Para cada tipo de conecção, uma conjunto diferente de dados poderá ser requisitado.
Por exemplo, para que uma conecção "Access" seja criada, é necessário a indicação do arquivo "mdb" e de um título para a conecção.

Janela de Login

Dependendo do banco de dados, um "Login" poderá ser requisitado, caso não hajam usuários cadastrados deixe os campos em branco e precione o botão "OK" que a conecção será estabelecida.

Visualizando uma Tabela

As mesmas regras para os "nós" da árvore de objetos das conecções estão valendo. Dependendo do tipo de objeto selecionado, poderão existir ações individuais ou coletivas.

Banco de Dados Padronizados

São banco de dados que possuem determinados tipos de dados, isto é, que possuem certos tipos de tabelas e estas tabelas possuem certos tipos de campos.

O Proceda sempre tentará enquadrar um banco de dados em um padrão, se nenhum padrão for reconhecido ele será enquadrado no padrão genérico e as ações sobre ele serão limitadas as ações genéricas.

Banco de Dados Padrão ANA-Hidro

São banco de dados que possuem pelo menos as seguintes tabelas:

  • Chuvas
  • Cotas
  • Vazoes
  • Series_Chuvas
  • Series_Cotas
  • Series_Vazoes

Reconhecimento Automático de Dados

Se um banco de dados for padronizado, certos tipos de informações poderão ser reconhecidas automaticamente, como por exemplo os postos de chuva ou de vazão de um banco de dados padrão ANA-Hidro. E com estas informações reconhecidas, determinados tipos de ações poderão ser disponibilizadas.

Reconhecimento de Postos

Como dito acima, sempre que possível o sistema tentará reconhecer os dados sobre os postos disponibilizados.

Conversão de Postos

Com os postos reconhecidos o sistema disponibilizará a ação de conversão automática para o formato interno do Proceda.





Após a conversão ser realizada, a conecção poderá ser liberada pelo usuário caso ele não necessite mais.
O arquivo "proceda" criado permanecerá em memória até ser salvo em disco pelo usuário.
